WHAT GRIDWOLF IS
================================================================
Passive ICS/SCADA network discovery + vulnerability intelligence
platform. Packaged for OVA, Docker Hub, Azure Marketplace, AWS
Marketplace, airgap, and an enterprise-appliance build.

Stack:
  backend/   FastAPI + SQLAlchemy async + Pydantic + JWT + SQLite (default)
  frontend/  React 19 + Vite + Tailwind 4.2 + Zustand + React Query
  deploy/    per-channel packaging (aws, azure, airgap, kubernetes, ova)

Port plan (appliance):
   22  SSH
  3000 Gridwolf UI
  8000 Gridwolf API
  9090 Cockpit web console

Key convention: NO demo / mock data in production code. Seed files,
vuln feeds, fingerprint DBs and export payloads are empty — repopulate
from backend at runtime. Frontend aligns to backend enums, not vice
versa. Status/severity/type color tokens are SEMANTIC — do not re-theme.

CI jobs (all must be green): lint-backend, lint-frontend, build-frontend.
`.gitattributes` forces LF on source files — ruff format --check is
fragile on Windows CRLF.

QUICK REFERENCE
================================================================

Build OVA — preferred (CI):
  git tag release/v1.1.0
  git push origin release/v1.1.0
  # Then watch:  gh run watch --exit-status
  # Artifact:    Actions → Build Gridwolf OVA → gridwolfOS-1.1.0
  # Release:     gh release view release/v1.1.0  (draft)

Build OVA — local / airgap rebuild:
  cd deploy/ova/packer
  packer init .
  packer build -var gridwolf_version=1.1.0 gridwolf.pkr.hcl
  # Output: build/gridwolf-<ver>-<yyyymmdd>.ova (assembled by
  # assemble-ova.sh — qemu-img + sed-rendered OVF + tar, no ovftool)

Manual provision on bare Ubuntu 24.04 VM:
  sudo GRIDWOLF_ADMIN_USERNAME=admin \
       GRIDWOLF_ADMIN_PASSWORD='...' \
       GRIDWOLF_VERSION=1.1.0 \
       ./deploy/ova/provision.sh

Run locally for manual test:
  docker compose up -d --build
  open http://localhost:3000

Frontend checks:
  cd frontend && npx tsc --noEmit && npm run lint && npm run build

Backend checks:
  cd backend && ruff check . && ruff format --check .

CI watch after push:
  gh run list --branch main --limit 1
  gh run watch <id> --exit-status
